# 测试大模型编程能力的有趣项目

## 入门级（单文件、逻辑驱动）

1. **生命游戏 (Conway's Game of Life)** — 测试网格逻辑、状态管理
2. **2048 游戏** — 测试矩阵操作、动画、交互逻辑
3. **贪吃蛇 / 俄罗斯方块** — 经典小游戏，考验实时渲染和碰撞检测
4. **Markdown 解析器** — 测试字符串处理、递归、AST 构建

## 中级（多模块、算法为主）

5. **解释器/计算器** — 实现一个小型编程语言或表达式求值器，测试解析和递归能力
6. **路径寻找可视化** — A*/Dijkstra 算法可视化，测算法实现和动画渲染
7. **简易数据库** — 支持 SQL 子集的内存数据库（SELECT/JOIN/WHERE）
8. **正则表达式引擎** — 从零实现正则匹配，NFA/DFA 构造

## 高级（系统设计、多文件协作）

9. **简易操作系统内核** — 在 RISC-V/x86 上实现最基础的 OS
10. **Redis 简易版** — 实现基本数据结构 + 网络协议 + 持久化
11. **自建 Web 框架** — 从 TCP socket 开始，实现 HTTP 解析 + 路由 + 模板引擎
12. **简易 Docker** — 用 Linux namespace/cgroup 实现容器隔离

## 创意型（趣味性强）

13. **物理模拟** — 弹簧、流体、布料模拟，考验数学和渲染
14. **音频可视化器** — Web Audio API + Canvas，测试多媒体处理
15. **光线追踪器 (Ray Tracer)** — 经典图形学项目，几百行代码出震撼效果
16. **代码高尔夫挑战** — 用最少的字符实现特定功能

## 推荐的测试方式

| 维度 | 适合测试的项目 |
|------|---------------|
| 算法能力 | A* 可视化、正则引擎、解释器 |
| 系统设计 | 简易数据库、Redis、Web 框架 |
| 前端/交互 | 2048、物理模拟、光线追踪 |
| 调试能力 | 给一个有 bug 的复杂项目让它修 |
| 创造力 | "用你喜欢的语言做一个让我惊讶的东西" |

## 社区流行的测试基准

- **SWE-bench**：让模型修真实 GitHub issue
- **HumanEval**：函数级编程题
- **给一个模糊需求**，看模型能不能主动澄清、设计方案、实现并测试

## 进阶挑战（高区分度，全面测试大模型真实能力）

以下题目侧重不同维度，能较好区分模型的真实编程水平。

### 题目一：表达式模板引擎（Python）

**测试维度**：解析能力、递归、代码生成、安全沙箱

实现一个模板引擎，支持以下特性：
- 变量插值：`{{ user.name }}`
- 条件渲染：`{% if score > 90 %} 优秀 {% elif score > 60 %} 及格 {% else %} 不及格 {% endif %}`
- 循环：`{% for item in items %} {{ item }} {% endfor %}`
- 过滤器管道：`{{ name | upper | truncate(20) }}`
- 模板继承：`{% extends "base.html" %}` + `{% block content %}...{% endblock %}`
- 安全性：禁止执行任意代码，只允许白名单操作

**复杂度来源**：需要手写词法分析 + 语法树构建 + 沙箱执行，涉及多层嵌套解析

### 题目二：简易 Git 版本控制系统（Python）

**测试维度**：系统设计、哈希树、文件系统操作、二进制协议

实现一个迷你 Git，支持以下命令：
```
mygit init                          # 初始化仓库
mygit add <file>                    # 暂存文件
mygit commit -m "message"           # 创建提交（SHA-1 哈希）
mygit log                           # 显示提交历史
mygit diff <commit1> <commit2>      # 比较两个版本的差异
mygit branch <name>                 # 创建分支
mygit checkout <branch>             # 切换分支
mygit merge <branch>                # 合并分支（处理冲突）
```

**复杂度来源**：需要实现 blob/tree/commit 对象模型、内容寻址存储、三方合并算法

### 题目三：正则表达式引擎（Python）

**测试维度**：自动机理论、图算法、回溯/NFA 转换

从零实现正则引擎（不使用 `re` 模块），支持：
- 基本匹配：`a`, `abc`
- 元字符：`.`, `^`, `$`
- 量词：`*`, `+`, `?`, `{m,n}`
- 字符类：`[a-z]`, `[^0-9]`
- 分组与捕获：`(ab)+`，支持 `\1` 反向引用
- 交替：`cat|dog`
- 实现方式：Thompson NFA 构造，要求 O(nm) 时间复杂度

**复杂度来源**：需要理解 Thompson 构造法，将正则表达式转为 NFA 再模拟执行

### 题目四：Raft 共识算法模拟（Python）

**测试维度**：分布式系统、并发编程、状态机、网络模拟

在单进程内模拟 Raft 共识协议：
- 实现Leader Election（随机超时 + 投票）
- 实现Log Replication（日志追加一致性）
- 模拟网络分区、节点宕机、消息延迟
- 用线程模拟 5 个节点，通过队列模拟网络通信
- 提供可视化输出展示选举和日志复制过程

**复杂度来源**：并发正确性、状态机转换、网络不确定性处理

### 题目五：CSV-SQL 查询引擎（Python）

**测试维度**：SQL 解析、查询优化、执行计划、类型系统

实现一个能对 CSV 文件执行 SQL 查询的引擎：
```sql
SELECT name, AVG(score) as avg_score
FROM students.csv
JOIN courses.csv ON students.id = courses.student_id
WHERE students.grade > 60
GROUP BY name
HAVING avg_score > 80
ORDER BY avg_score DESC
LIMIT 10
```

支持：SELECT（含聚合函数、别名）、FROM（多表）、JOIN（INNER/LEFT）、WHERE、GROUP BY、HAVING、ORDER BY、LIMIT、子查询

**复杂度来源**：需要 SQL 词法/语法解析器、查询计划构建、多表 JOIN 实现、聚合计算

### 题目六：光线追踪渲染器（Python）

**测试维度**：数学（线性代数/几何）、物理模拟、算法优化

实现一个能输出 PPM/PNG 图片的光线追踪器：
- 球体、平面、三角形求交
- Phong 光照模型（环境光 + 漫反射 + 高光）
- 阴影投射
- 镜面反射（递归光线弹射）
- 折射（玻璃材质，Snell 定律）
- 抗锯齿（多重采样）
- 多线程加速渲染

**复杂度来源**：大量向量运算、递归光线追踪、物理公式实现、性能优化

### 题目七：Mini Redis（Go）

**测试维度**：网络编程、协议实现、并发安全、持久化

用 Go 实现一个兼容 Redis 协议（RESP）的内存数据库：
- 支持 String、List、Hash、Set 数据类型
- 支持 PUBLISH/SUBSCRIBE 发布订阅
- 支持 TTL 过期机制
- 实现 RDB 快照持久化
- 支持 AOF（Append Only File）日志
- TCP 服务器，能被标准 `redis-cli` 连接

**复杂度来源**：RESP 协议解析、并发数据结构、发布订阅模式、持久化编码
